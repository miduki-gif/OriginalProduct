# オリジナルプロダクトテーマ
 - 要件の決まり具合を見える化し、クライアントとの合意形成を助けるアプリ
 ## ターゲット顧客
 - 受託開発を行っている企業
   (案件の数重視ではなく長期で関係を大事にしている)
 - 開発を請け負っているフリーランスの方、
 - 曖昧なことにより後々トラブルが発生するのを嫌に思うPM等
## プロダクト名
 - FramView
## カテゴリ
 - 要件定義合意支援ツール
 （大項目に分けるとソフトウェア開発支援ツール）
## 課題
 - アプリやシステム開発を行う際に後から追加や変更が発生したり、
どこからが追加や変更の範囲かが曖昧になりトラブルや工数が余分に発生してしまうこと
## アプリによって解決できること
 - 要件ごとに決まっていること決まっていないことを項目単位で視覚的に可視化できる
 - 質問に答えるだけでどんな要件が必要か整理される
 - 追加・変更の影響範囲を初期段階で視覚的に把握できる
 - 関係者間でどこからが未確定かの共通認識を持った状態で要件定義を進められ、
 　後々のトラブルや認識違いを減らす助けになる
 - 追加や変更が発生した際に客観的に追加での対応になること等客観的な情報から説明することができる
## キャッチコピー
 - 要件の決定度を可視化
## できること
 - どんなアプリやシステムが作りたいのか質問事項に対して選択形式で選択し(オプション等はyes,no)、
 やりたいことを引き出す
 - 選択してもらった回答で必要なものの機能が一覧で表示
 - 機能一覧で表示した機能をプレビュー画面で簡易的に確認できる
 - 各項目の決定状況と要件決定度をステータスで表示できる
## どんな機能がユーザ側で必要か
 - どんなことをしたいのかをヒアリングするために以下のパターンに分けて質問をする
  - 既存業務を改善・アップデートしたい
   - 新規事業や新しい取り組みをしたい
  - 既存業務の場合
   - 新しい取り組みや新規事業の場合

## 競合サービス
 - Miro
  - ワイヤーフレームなどの画面の枠組みを作成することができる
  - 矢印や付箋など要件整理や思考整理を行うことができる
  - 決定度やステータス（仮や確定）がテキストでは表示できるが、データの形で見えない
 - Notion
  - テーブル形式でデータを管理できる
  - 優先度や進捗ステータスをデータで持つことができる
  - やることをカンバンやテーブル形式で表示することができる
  - 進捗ステータスはデータで管理できるが、保留や未確定になぜなっているかが
    全体を見渡して管理できる構造になっていない
 - Excel
  - フロー図を記述することや決定度やステータス(仮や確定)をテキストやセルに色を付けて
    見た目上で管理は出来るが、各項目ごとで決定度を全体として構造化された状態で確認できない
